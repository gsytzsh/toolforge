About URL Extractor

This URL extractor scans any text and finds all HTTP and HTTPS links using regex pattern matching. Results are automatically deduplicated, providing a clean list of unique URLs for further processing or analysis.

URL Pattern Explained

The regex matches standard web URL formats:

Pattern: https?://[^\s"'<>()]+

Breakdown:
https?      → http or https (s is optional)
://         → Protocol separator
[^\s"'<>()]+ → One or more characters that aren't
               whitespace, quotes, or brackets

Limitations

  • HTTP/HTTPS only: FTP, mailto, tel, and other protocols not supported
  • No relative URLs: Paths like /page aren't extracted
  • No validation: Doesn't check if URLs are accessible
  • Parentheses: URLs ending in ) may include trailing paren
  • Encoded URLs: Punycode international domains may not match perfectly

Frequently Asked Questions

How does URL extraction work?
URL extraction uses regex patterns to match HTTP and HTTPS URLs. The pattern looks for the protocol (http:// or https://), domain name, and optional path/query parameters. All matches are collected and deduplicated.
What URL formats are supported?
Full URLs with http:// or https:// protocol. Supports subdomains, paths, query strings, fragments, and port numbers. Examples: https://example.com, http://api.domain.com:8080/v1/users?id=123
Are relative URLs extracted?
No, only absolute URLs with http:// or https:// protocol are extracted. Relative URLs like /page/path or //cdn.example.com are not matched. This ensures only complete, usable URLs are returned.
Is the output deduplicated?
Yes, duplicate URLs are automatically removed. Each unique URL appears only once in the output, regardless of how many times it appears in the input text.
What about FTP or other protocols?
This tool only extracts HTTP and HTTPS URLs. Other protocols like FTP, mailto, tel, or file:// are not supported. Most web-related use cases only require HTTP/HTTPS links.
