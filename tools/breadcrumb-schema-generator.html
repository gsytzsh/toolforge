关于 Breadcrumb Schema 生成器

此面包屑 Schema 生成器可创建符合 schema.org 标准的 BreadcrumbList JSON-LD 结构化数据。它帮助搜索引擎理解网站的层级导航结构，可能在 Google 搜索结果中显示更友好的面包屑导航而非完整 URL。

BreadcrumbList 结构说明

  1. BreadcrumbList：根类型，包含整个面包屑列表
  2. itemListElement：数组，包含所有面包屑项
  3. ListItem：每个面包屑项的类型
  4. position：该项在路径中的位置（从 1 开始）
  5. name：显示给用户的标签文本
  6. item：该项指向的完整 URL
生成的 JSON-LD 示例：

{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "首页",
      "item": "https://example.com/"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "博客",
      "item": "https://example.com/blog/"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "技术文章",
      "item": "https://example.com/blog/tech/"
    }
  ]
}

输入格式说明

每行输入一个面包屑项，使用 Name :: URL 格式：

输入示例：

首页 :: https://example.com/
产品分类 :: https://example.com/products/
电子产品 :: https://example.com/products/electronics/
手机 :: https://example.com/products/electronics/phones/

每行结构：标签名 :: 完整 URL
顺序：从首页到当前页面

SEO 最佳实践

要点 说明
与页面导航一致 结构化数据必须与页面上可见的面包屑链接匹配
使用绝对 URL item 字段必须是完整的 URL（包含协议）
至少两项 BreadcrumbList 必须包含至少两个 ListItem
position 连续 位置编号必须从 1 开始且连续递增
首页可省略 URL 首页的 item 字段可以省略

常见应用场景

  • 电商网站：展示分类层级（首页 > 分类 > 子分类 > 产品）
  • 博客/新闻：文章路径（首页 > 栏目 > 文章标题）
  • 文档站点：文档层级（首页 > 章节 > 页面）
  • 论坛：版块导航（首页 > 大区 > 版块 > 主题）
  • 企业网站：页面层级（首页 > 关于我们 > 团队介绍）

常见错误及避免方法

错误 影响 解决方案
缺少 position 验证失败 为每个 ListItem 添加 position
URL 使用相对路径 Google 可能忽略 使用完整绝对 URL
只有一项 不符合要求 至少包含两项
与页面导航不一致 可能被降权 保持结构化数据与可见导航一致

如何使用面包屑 Schema 生成器

  1. 输入面包屑项：每行输入一个项，格式为"标签名 :: URL"。
  2. 确认顺序：从首页开始，依次到当前页面。
  3. 点击生成：工具会生成符合 schema.org 标准的 JSON-LD 代码。
  4. 复制使用：将生成的代码放入网页的<head>或<body>中。
  5. 验证测试：使用 Google Rich Results Test 验证部署效果。

提示

  • 每行必须包含"::"分隔符，前面是标签，后面是 URL
  • URL 必须是完整格式（包含 https://）
  • position 会自动从 1 开始递增
  • 至少需要输入两个面包屑项
  • 最后一个 URL 可以是当前页面，也可以省略

常见问题解答

什么是 Breadcrumb Schema（面包屑模式）？
Breadcrumb Schema 是 schema.org 定义的 BreadcrumbList 结构化数据类型，用于向搜索引擎展示网站的层级导航路径。它由 itemListElement 数组组成，每个 ListItem 包含 position（位置）、name（名称）和 item（URL）。
为什么要在 SEO 中使用面包屑结构化数据？
Google 使用面包屑结构化数据来理解页面在网站层级中的位置，可能在搜索结果中显示面包屑导航而非完整 URL。这能提升点击率、改善用户体验，并帮助搜索引擎更好地抓取网站结构。
面包屑结构化数据的正确格式是什么？
必须使用 BreadcrumbList 类型，itemListElement 数组包含至少两个 ListItem。每个 ListItem 必须有 position（从 1 开始的整数）、name（面包屑标签）和 item（完整 URL，homepage 除外可省略）。
面包屑的顺序应该如何排列？
从左到右（或从首页到当前页）：position 1 是首页，position 2 是父级分类，最后是当前页面。例如：首页 > 博客 > 技术文章 > 具体文章。最后一个 item 可以是当前页面 URL 或省略。
如何在网页中部署面包屑结构化数据？
将生成的 JSON-LD 代码放入页面的 <head> 或 <body> 中。确保与页面上可见的面包屑导航保持一致。部署后用 Google Rich Results Test 或 Schema Markup Validator 验证。
面包屑结构化数据的常见错误有哪些？
常见错误：缺少 required 字段（position、name）、URL 不是绝对路径、itemListElement 少于两项、position 不连续、结构化数据与页面实际导航不一致。这些会导致 Google 忽略或降权处理。
