About Naming Convention Converter

This converter transforms text between different naming conventions used in programming: snake_case, camelCase, PascalCase, and kebab-case. Each convention has specific use cases across different languages and frameworks.

Conversion Algorithms

snake_case to camelCase:

1. Split by underscore: "user_name" → ["user", "name"]
2. Keep first word lowercase
3. Capitalize remaining words: "name" → "Name"
4. Join: "user" + "Name" = "userName"

camelCase to snake_case:

1. Find uppercase letters: "userName" → "user" + "N" + "ame"
2. Insert underscore before each uppercase
3. Convert to lowercase: "User" → "user"
4. Join: "user_name"

Frequently Asked Questions

What is camelCase and when is it used?
camelCase starts with lowercase letter, then capitalizes each subsequent word (e.g., userName, totalCount). It's the standard convention in JavaScript, Java, C#, and JSON APIs for variable names and object properties.
What is snake_case and when is it used?
snake_case uses lowercase letters with underscores between words (e.g., user_name, total_count). It's common in Python, Ruby, SQL databases, and environment variables. Preferred for readability in many languages.
What is PascalCase and how does it differ from camelCase?
PascalCase (UpperCamelCase) capitalizes the first letter of every word including the first (e.g., UserName, TotalCount). Unlike camelCase which starts lowercase. Used for class names in most languages and component names in React.
What is kebab-case and where is it used?
kebab-case uses lowercase with hyphens between words (e.g., user-name, total-count). Common in HTML attributes (data-user-id), CSS class names, URL slugs, and configuration files. Not valid in most programming languages.
How do I convert JSON keys between conventions?
Paste the JSON and convert keys individually, or use this tool for single keys. For bulk conversion, use code: JavaScript's Object.keys().map() with regex replacement, or libraries like camelcase-keys for automatic conversion.
