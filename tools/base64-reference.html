About Base64 Reference

This Base64 reference table shows the complete character set used in Base64 encoding. Each of the 64 characters corresponds to a 6-bit value (0-63), enabling binary-to-text conversion.

Base64 Character Set Breakdown

Characters Indices Count
A-Z (uppercase) 0-25 26
a-z (lowercase) 26-51 26
0-9 (digits) 52-61 10
+ (plus) 62 1
/ (slash) 63 1

How Base64 Encoding Works

Base64 converts binary data to ASCII text using these steps:

  1. Group bytes: Input is split into groups of 3 bytes (24 bits)
  2. Split to 6-bit: Each 24-bit group becomes four 6-bit values
  3. Map to characters: Each 6-bit value (0-63) maps to a Base64 character
  4. Add padding: If input isn't divisible by 3, add = padding
Example: Encoding "ABC" (ASCII: 65, 66, 67)

Binary:  01000001 01000010 01000011
Grouped: 010000 010101 000010 01000011
         |      | |      | |      | |      |
         16     21 2      17
Chars:   Q      Y  B      R

Result:  "QUJD" (4 Base64 characters from 3 input bytes)

URL-Safe Base64 (Base64URL)

Standard Base64 uses + and / which have special meaning in URLs. Base64URL replaces these characters:

Variant Char 62 Char 63 Padding Use Case
Standard Base64 + / Required General purpose
Base64URL - _ Optional URLs, JWT, OAuth

Size Overhead

Base64 encoding increases data size:

  • Ratio: 4 output characters per 3 input bytes
  • Overhead: Approximately 33.3% increase
  • Formula: encodedSize = ceil(originalSize / 3) × 4
  • With padding: Up to 2 padding characters may be added
Input Size    → Output Size (approx)
100 bytes     → 136 bytes
1 KB          → 1.36 KB
100 KB        → 136 KB
1 MB          → 1.36 MB

Common Applications

  • Data URLs: Embed images in HTML/CSS (data:image/png;base64,...)
  • Email: MIME encoding for attachments (quoted-printable, base64)
  • JWT tokens: Encode header and payload using Base64URL
  • API authentication: HTTP Basic Auth (Authorization: Basic base64(user:pass))
  • Binary in JSON/XML: Include binary data in text-based formats

Frequently Asked Questions

What is the Base64 character set?
Base64 uses 64 characters: uppercase A-Z (indices 0-25), lowercase a-z (indices 26-51), digits 0-9 (indices 52-61), plus + (index 62), and forward slash / (index 63). Padding uses = character.
How does Base64 encoding work?
Base64 processes binary data in 6-bit groups. Each 6-bit value (0-63) maps to a character in the Base64 alphabet. Three bytes (24 bits) become four Base64 characters. Padding (=) is added if input isn't divisible by 3.
What is URL-safe Base64?
URL-safe Base64 (Base64URL) replaces + with - and / with _ to make the output safe for URLs and filenames. It's commonly used in JWT tokens and OAuth parameters. Padding is often omitted.
Why does Base64 use 6 bits per character?
Six bits provides exactly 64 values (2^6 = 64), matching the 64-character alphabet. This allows efficient encoding: 3 bytes (24 bits) encode to exactly 4 Base64 characters with no waste.
What is Base64 padding and when is it used?
Padding (=) is added to make the Base64 output length a multiple of 4. One = means 2 bytes of input (16 bits → 3 chars + padding). Two = means 1 byte of input (8 bits → 2 chars + padding).
